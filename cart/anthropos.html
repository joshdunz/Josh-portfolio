Design-process

01 Identify Key Issues

I found it tedious to scroll when I had topic in mind. I didn't like the feeling of not knowing immediately how much I'm spending. I wanted my products to feature scent notes so I know how they smelt generally.

02 Find Best Practices

Referencing the Sephora, Alibaba and Macy's ecommerce stores, I compiled a table of shared features. These were Item manufacturer Info, the number of items per page and a 5-star review format.

03 Implement Design

With these features in mind, I coded a basic structure of a webpage in react, adding functionality like cards which pulled from a data.js file and implementing the cart function. I then added simple buttons. Once the basic page was completed, I proceeded to implement a cohesive color scheme and design the outlook of the page.

Feature 1

Filter & Sort

Problem

It's incredibly tedious to scroll through all products when you already have an idea of what you want. (Price, Smell, Brand)

Aesthetic note

I chose to put it on the left because I wanted to prevent the options menu from spilling over the product cards, covering them. This would have been the case if it had been positioned on top.

Solution

I built a filtering property which allows users to choose a product by price range, scent note, or brand

Feature 2

Cart

Problem

Cart contents are typically hidden from view as dark pattern to keep users buying without knowing hwo much they're spending.

Solution

I made the cart part of the homepage so users can see how much they're spending. The remove button is also obvious and readily accessible. Items can be seen, added and removed easily. There is also a sum total at the bottom to enable users to keep track of their bills
